## 1. 제품 개요

vibe-pdf는 사용자가 파일을 서버에 업로드하지 않고 브라우저에서 직접 PDF를 병합(Merge), 분할(Split), 페이지 추출(Extract), 페이지 삭제(Delete), 페이지 회전(Rotate), 페이지 순서 변경(Reorder), 암호화(Password Protect), 이미지를 PDF로 변환(Image to PDF)할 수 있는 클라이언트 사이드 전용 웹 애플리케이션이다.

| 항목 | 내용 |
|------|------|
| 제품명 | vibe-pdf |
| 플랫폼 | 웹 브라우저 (데스크톱 우선) |
| 기술 스택 | React + TypeScript + Vite + pdf-lib |
| 배포 방식 | 정적 파일 호스팅 (서버 없음) |
| MVP 기능 | PDF 병합, PDF 스플릿, PDF 페이지 추출, PDF 페이지 삭제, PDF 페이지 회전, PDF 페이지 순서 변경, PDF 암호화, 이미지를 PDF로 변환 (8개) |

## 2. 문제 정의

### 현재 사용자가 겪는 문제

1. **프라이버시 위협**: 기존 PDF 도구(Smallpdf, iLovePDF 등)는 파일을 서버에 업로드한다. 민감한 문서(계약서, 개인 정보 포함 문서)를 제3자 서버로 보내야 한다.
2. **불필요한 복잡성**: 많은 PDF 도구가 OCR, 압축, 서명, 변환 등 불필요한 기능으로 인터페이스가 복잡하다.
3. **속도 저하**: 서버 업로드/처리/다운로드 왕복으로 처리 시간이 길다.
4. **비용**: 대부분의 고품질 도구는 유료이거나 기능을 제한한다.
5. **페이지 단위 편집 불가**: 특정 페이지만 뽑거나, 불필요한 페이지를 제거하는 간단한 작업도 전용 소프트웨어가 필요하다.
6. **스캔 방향 오류**: 스캔된 문서의 일부 페이지가 잘못된 방향으로 저장되어 있어도 간단히 회전할 수 없다.
7. **페이지 순서 수정 불가**: PDF 내 페이지 순서가 잘못되었을 때 재배열하려면 별도 소프트웨어가 필요하다.
8. **PDF 보안 적용 불가**: 민감한 PDF를 공유할 때 비밀번호를 설정하려면 별도 소프트웨어나 서버 기반 도구가 필요하다.
9. **이미지를 PDF로 변환 불가**: 스캔 이미지나 사진을 PDF로 묶으려면 별도 소프트웨어나 서버 기반 변환 도구가 필요하다.

### 핵심 통찰

"이미지를 PDF로 변환하는 작업도 서버 없이 브라우저에서 충분히 처리 가능하다. 파일이 내 장치를 떠나지 않아도 된다."

## 3. 목표

### 제품 목표

- 파일이 서버로 전송되지 않는 프라이버시 보장 PDF 도구 제공
- PDF 8가지 기능을 최소 클릭으로 완료 가능한 단순 UX 구현
- 설치 없이 브라우저에서 즉시 사용 가능한 접근성 확보

### MVP 범위 목표

- PDF 병합: 2개 이상의 PDF 파일을 하나로 합치기
- PDF 스플릿: 하나의 PDF 파일을 페이지 단위로 분리하기
- PDF 페이지 추출 (Extract): 원하는 페이지만 골라 새 PDF로 저장하기
- PDF 페이지 삭제 (Delete): 불필요한 페이지를 제거하고 저장하기
- PDF 페이지 회전 (Rotate): 특정 페이지를 90/180/270도 회전하여 저장하기
- PDF 페이지 순서 변경 (Reorder): 드래그앤드롭으로 페이지 순서를 변경하여 저장하기
- PDF 암호화 (Password Protect): user password를 설정하여 AES-256으로 암호화된 PDF 저장하기
- 이미지를 PDF로 변환 (Image to PDF): JPG/PNG 등 이미지 파일을 페이지로 배치하여 단일 PDF로 저장하기

## 4. 대상 사용자

### Primary User: 일반 직장인 / 사무직

- **특징**: PDF 편집 전문가가 아님. 가끔 PDF 합치기/나누기/페이지 편집이 필요함
- **상황**: 계약서, 보고서, 스캔 서류 등 민감한 문서를 다룸
- **니즈**: 빠르고 단순하게 처리. 파일이 외부로 나가지 않기를 원함
- **기기**: 데스크톱 / 노트북 (크롬, 엣지, 파이어폭스)

### Secondary User: 학생 / 프리랜서

- **특징**: 비용에 민감. 무료 도구 선호
- **상황**: 과제 제출용 PDF 합치기, 포트폴리오 PDF 편집, 스캔 문서 정리, 사진을 PDF로 제출
- **니즈**: 무료, 빠름, 설치 불필요

## 5. 핵심 기능

### Feature 1: PDF 병합 (Merge)

여러 PDF 파일을 사용자가 지정한 순서대로 하나의 PDF로 합친다.

### Feature 2: PDF 스플릿 (Split)

하나의 PDF 파일을 선택한 페이지 범위 또는 각 페이지별로 분리하여 다운로드한다.

### Feature 3: PDF 페이지 추출 (Extract)

하나의 PDF 파일에서 원하는 특정 페이지만 선택하여 새로운 PDF 파일로 저장한다.

### Feature 4: PDF 페이지 삭제 (Delete)

하나의 PDF 파일에서 불필요한 특정 페이지를 제거하고 나머지 페이지로 구성된 새 PDF 파일로 저장한다.

### Feature 5: PDF 페이지 회전 (Rotate)

하나의 PDF 파일에서 특정 페이지를 선택하여 90도/180도/270도 단위로 회전하고 저장한다.

### Feature 6: PDF 페이지 순서 변경 (Reorder)

하나의 PDF 파일의 페이지 순서를 드래그앤드롭으로 변경하고 새 PDF 파일로 저장한다.

### Feature 7: PDF 암호화 (Password Protect)

암호화되지 않은 PDF 파일에 user password(열기 비밀번호)를 설정하여 AES-256으로 암호화한다.

### Feature 8: 이미지를 PDF로 변환 (Image to PDF)

JPG, PNG, WEBP, GIF 등의 이미지 파일 1개 이상을 업로드하여 각 이미지를 한 페이지로 배치한 단일 PDF 파일로 변환한다. 이미지 순서를 드래그앤드롭으로 변경할 수 있으며 페이지 크기 옵션(A4, Letter, 이미지 원본 크기)을 선택할 수 있다.

### 6.7 PDF 암호화 (Password Protect)

| ID | 요구사항 |
|----|----------|
| PP-01 | 사용자는 암호화되지 않은 PDF 파일 1개를 선택할 수 있어야 한다 |
| PP-02 | 파일은 드래그 앤 드롭 또는 파일 선택 다이얼로그로 추가할 수 있어야 한다 |
| PP-03 | 업로드된 PDF의 파일명과 전체 페이지 수를 표시해야 한다 |
| PP-04 | user password(열기 비밀번호) 입력 필드를 제공해야 한다 |
| PP-05 | 비밀번호 확인 입력 필드를 제공해야 하며, 두 입력값이 일치해야 암호화 버튼이 활성화된다 |
| PP-06 | 비밀번호 입력 필드는 기본적으로 마스킹(●●●) 처리되어야 하며, 사용자가 표시/숨김을 전환할 수 있어야 한다 |
| PP-07 | 비밀번호는 최소 8자 이상을 권장하며, 8자 미만 입력 시 "8자 이상 권장" 안내 메시지를 표시해야 한다 (입력 차단은 하지 않음) |
| PP-08 | 암호화 알고리즘은 AES-256을 사용해야 한다. RC4, 40-bit 등 약한 암호화는 사용하지 않는다 |
| PP-09 | owner password는 UI에 노출하지 않고 내부적으로 생성하여 적용해야 한다 |
| PP-10 | 암호화 실행 버튼 클릭 시 브라우저에서 PDF를 암호화하여 다운로드 파일로 제공해야 한다 |
| PP-11 | 출력 파일명은 기본값 `{원본파일명}_protected.pdf`이며 사용자가 변경할 수 있어야 한다 |
| PP-12 | 처리 중 로딩 상태를 표시해야 한다 |
| PP-13 | 모든 처리는 브라우저(클라이언트) 내에서만 수행되어야 한다. 파일과 비밀번호는 서버로 전송되지 않아야 한다 |
| PP-14 | 처리 완료 후 비밀번호 데이터의 메모리 참조를 해제해야 한다 |
| PP-15 | 비밀번호 및 파일 데이터를 localStorage, sessionStorage, IndexedDB에 저장해서는 안 된다 |
| PP-16 | 이미 암호화된 PDF를 업로드하면 오류 메시지를 표시하고 처리를 거부해야 한다 |

### 6.8 이미지를 PDF로 변환 (Image to PDF)

| ID | 요구사항 |
|----|----------|
| IP-01 | 사용자는 JPG, PNG, WEBP, GIF 형식의 이미지 파일을 1개 이상 선택할 수 있어야 한다 |
| IP-02 | 파일은 드래그 앤 드롭 또는 파일 선택 다이얼로그로 추가할 수 있어야 한다 |
| IP-03 | 추가된 이미지 목록에 각 이미지의 파일명과 미리보기(썸네일)를 표시해야 한다 |
| IP-04 | 이미지 카드를 드래그앤드롭으로 순서를 변경할 수 있어야 한다 (각 이미지가 PDF의 한 페이지가 된다) |
| IP-05 | 각 이미지를 개별적으로 목록에서 제거할 수 있어야 한다 |
| IP-06 | 페이지 크기 옵션을 선택할 수 있어야 한다: A4, Letter, 이미지 원본 크기 |
| IP-07 | 선택된 이미지 순서대로 각 이미지를 한 페이지로 배치한 단일 PDF 파일로 출력되어야 한다 |
| IP-08 | 이미지는 선택한 페이지 크기에 맞게 비율을 유지하며 중앙 배치(fit)되어야 한다 |
| IP-09 | 출력 파일명은 기본값 `images.pdf`이며 사용자가 변경할 수 있어야 한다 |
| IP-10 | 처리 중 로딩 상태를 표시해야 한다 |
| IP-11 | 모든 처리는 브라우저(클라이언트) 내에서만 수행되어야 한다 |

### 6.9 공통 요구사항

| ID | 요구사항 |
|----|----------|
| C-01 | 지원하지 않는 파일 형식 업로드 시도 시 명확한 오류 메시지를 표시해야 한다 |
| C-02 | 업로드 전 파일 크기 경고를 표시해야 한다 (100MB 초과 시) |
| C-03 | 각 기능 완료 후 결과를 초기화하거나 새 작업을 시작할 수 있어야 한다 |
| C-04 | 모바일 브라우저에서도 기본 기능이 동작해야 한다 (반응형) |

## 7. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 프라이버시 | 파일 데이터는 브라우저 외부로 전송되지 않아야 한다. 네트워크 요청 0건 |

| 데이터 잔류 없음 | Password Protect 처리 후 비밀번호/파일 데이터가 localStorage, sessionStorage, IndexedDB에 남지 않아야 한다 |

## 8. 제외 범위 (Out of Scope)

MVP에서 명시적으로 제외하는 기능:

- OCR (텍스트 인식)
- PDF 압축 / 최적화
- 비밀번호 해제 (암호화 해제)
- owner password UI 노출 및 사용자 직접 설정
- 기존 암호화 PDF 재암호화
- 권한 세부 설정 (인쇄 제한, 복사 제한 등)
- 일괄 암호화 (여러 파일 동시 암호화)
- 워터마크 추가 / 제거
- PDF → 이미지/Word 변환
- 클라우드 저장 (Google Drive, Dropbox 등)
- 사용자 계정 / 인증
- 히스토리 / 최근 파일
- PDF 미리보기 썸네일 (Reorder 탭 제외)
- 주석 / 서명
- SVG, BMP, TIFF 등 비주류 이미지 형식 (Image to PDF)
- 이미지 편집 (크롭, 필터, 보정 등)
- 개별 페이지 여백/마진 설정

## 10. Acceptance Criteria

### 10.1 PDF 병합 AC

- [ ] AC-M-01: 사용자가 2개의 PDF를 추가하고 병합 버튼을 클릭하면 단일 PDF가 다운로드된다
- [ ] AC-M-02: 사용자가 3개의 PDF를 추가하고 순서를 변경한 후 병합하면 변경된 순서대로 합쳐진다
- [ ] AC-M-03: 파일 목록에서 특정 파일 제거 후 병합하면 해당 파일은 포함되지 않는다
- [ ] AC-M-04: 출력 파일명을 `report.pdf`로 변경하면 `report.pdf`로 다운로드된다
- [ ] AC-M-05: 파일 목록에 각 PDF의 파일명과 페이지 수가 표시된다
- [ ] AC-M-06: 병합 처리 중 로딩 인디케이터가 표시되고 완료 시 사라진다
- [ ] AC-M-07: PDF가 아닌 파일(.jpg, .docx 등)을 추가하려 하면 오류 메시지가 표시되고 목록에 추가되지 않는다
- [ ] AC-M-08: 파일 1개만 있는 상태에서 병합 버튼은 비활성화(disabled) 상태이다
- [ ] AC-M-09: 병합 완료 후 '새 작업 시작' 버튼을 클릭하면 파일 목록이 초기화된다

### 10.2 PDF 스플릿 AC

- [ ] AC-S-01: PDF를 업로드하면 파일명과 전체 페이지 수가 표시된다
- [ ] AC-S-02: '모든 페이지 개별 분리' 모드 선택 후 실행하면 페이지 수만큼의 PDF가 ZIP으로 다운로드된다
- [ ] AC-S-03: 범위 `1-3`을 입력하면 1, 2, 3페이지로 구성된 PDF 1개가 다운로드된다
- [ ] AC-S-04: 범위 `1-2, 4`를 입력하면 2개의 PDF(1-2페이지, 4페이지)가 ZIP으로 다운로드된다

### 10.7 PDF 암호화 AC (Password Protect)

- [ ] AC-PP-04: 표시/숨김 전환 버튼을 클릭하면 비밀번호 입력값이 평문으로 표시되거나 다시 마스킹된다

- [ ] AC-PP-07: 비밀번호와 비밀번호 확인 값이 일치하면 암호화 버튼이 활성화된다
- [ ] AC-PP-08: 비밀번호 입력 필드가 비어있으면 암호화 버튼은 비활성화 상태이다
- [ ] AC-PP-09: 암호화 버튼을 클릭하면 AES-256으로 암호화된 PDF가 다운로드된다
- [ ] AC-PP-10: 다운로드된 PDF를 Chrome, Firefox, Adobe Acrobat에서 열면 비밀번호를 요구한다
- [ ] AC-PP-11: 설정한 비밀번호를 입력하면 PDF가 정상적으로 열린다
- [ ] AC-PP-12: 잘못된 비밀번호를 입력하면 PDF가 열리지 않는다
- [ ] AC-PP-13: 출력 파일명 기본값이 `{원본파일명}_protected.pdf` 형태로 표시된다

### 10.8 이미지를 PDF로 변환 AC (Image to PDF)

- [ ] AC-IP-05: 개별 이미지 삭제 버튼을 클릭하면 해당 이미지가 목록에서 제거된다

- [ ] AC-IP-10: 출력 파일명을 `photos.pdf`로 변경하면 `photos.pdf`로 다운로드된다
- [ ] AC-IP-11: 이미지 파일이 없는 상태에서 변환 버튼은 비활성화 상태이다
- [ ] AC-IP-12: 변환 처리 중 로딩 인디케이터가 표시되고 완료 시 사라진다
- [ ] AC-IP-13: 변환 완료 후 '새 작업 시작' 버튼을 클릭하면 초기 상태로 돌아간다
- [ ] AC-IP-14: 네트워크 탭에서 이미지 데이터가 외부로 전송되는 요청이 없다

| E-D-06 | 1페이지 PDF에서 1페이지 삭제 시도 | "모든 페이지를 삭제할 수 없습니다" 오류 표시 |
| E-D-07 | 비밀번호로 잠긴 PDF | 처리 시 오류 메시지 표시 |
| E-D-08 | 손상된 PDF 파일 | 처리 시 오류 메시지 표시 |
| E-D-09 | 출력 파일명이 비어있음 | 기본값 `{원본파일명}_deleted.pdf` 사용 |

### 11.5 PDF 페이지 회전 Edge Cases (Rotate)

| # | 시나리오 | 기대 동작 |
|---|----------|-----------|
| E-R-01 | 범위를 비워두면 전체 페이지에 회전 적용 | 전체 페이지 회전 처리 |
| E-R-02 | 범위가 전체 페이지를 초과 (예: 5페이지 PDF에 `8` 입력) | 유효하지 않은 범위 오류 표시 |
| E-R-03 | 범위 형식 오류 (예: `a-b`, `1--3`) | 입력 즉시 유효성 검사 오류 표시 |
| E-R-04 | `0` 또는 음수 페이지 번호 입력 | 유효성 검사 오류 표시 |
| E-R-05 | 이미 회전된 페이지에 추가 회전 적용 | 기존 회전 각도에 선택 각도를 누적하여 적용 (예: 기존 90도 + 90도 = 180도) |
| E-R-06 | 비밀번호로 잠긴 PDF | 처리 시 오류 메시지 표시 |
| E-R-07 | 손상된 PDF 파일 | 처리 시 오류 메시지 표시 |
| E-R-08 | 출력 파일명이 비어있음 | 기본값 `{원본파일명}_rotated.pdf` 사용 |
| E-R-09 | 1페이지 PDF | 정상 처리, 해당 페이지 회전 |

### 11.6 PDF 페이지 순서 변경 Edge Cases (Reorder)

| # | 시나리오 | 기대 동작 |
|---|----------|-----------|
| E-RO-01 | 1페이지 PDF | 드래그앤드롭 비활성화, "순서 변경할 페이지가 없습니다" 안내 |
| E-RO-02 | 순서를 변경하지 않고 저장 버튼 클릭 | 원본과 동일한 PDF 다운로드 허용 |
| E-RO-03 | 드래그앤드롭 중 화면 스크롤 필요 (페이지 수 많음) | 스크롤 중 드래그 유지 지원 |
| E-RO-04 | 비밀번호로 잠긴 PDF | 처리 시 오류 메시지 표시 |
| E-RO-05 | 손상된 PDF 파일 | 처리 시 오류 메시지 표시 |
| E-RO-06 | 페이지 수가 매우 많음 (100페이지 이상) | 처리 허용, 카드 목록 스크롤 가능하게 렌더링 |
| E-RO-07 | 출력 파일명이 비어있음 | 기본값 `{원본파일명}_reordered.pdf` 사용 |
| E-RO-08 | Reset 후 다시 순서 변경 | 원본 순서에서 다시 드래그앤드롭 가능 |

### 11.7 PDF 암호화 Edge Cases (Password Protect)

| # | 시나리오 | 기대 동작 |
|---|----------|-----------|
| E-PP-01 | 이미 암호화된 PDF 업로드 | 업로드 즉시 "이미 보호된 PDF입니다. 암호화되지 않은 파일만 지원합니다" 오류 표시, 처리 거부 |
| E-PP-02 | 비밀번호 입력 후 확인란이 비어있는 상태 | 암호화 버튼 비활성화 유지 |
| E-PP-03 | 비밀번호와 확인이 일치했다가 한쪽을 수정하여 불일치 | 즉시 오류 메시지 표시, 암호화 버튼 비활성화 |
| E-PP-04 | 비밀번호로 공백 문자만 입력 | 유효하지 않은 비밀번호로 처리, 암호화 버튼 비활성화 |
| E-PP-05 | 비밀번호 8자 미만 입력 (예: `abc123`) | "8자 이상 권장" 안내 표시, 버튼은 비활성화 안 함 (확인란과 일치하면 진행 가능) |
| E-PP-06 | 매우 긴 비밀번호 입력 (100자 이상) | 처리 허용 (pdf-lib 제한 범위 내) |
| E-PP-07 | 손상된 PDF 파일 | 업로드 시 "유효하지 않은 PDF 파일입니다" 오류 메시지 표시 |
| E-PP-08 | 처리 실패 (pdf-lib 내부 오류) | "PDF 암호화 중 오류가 발생했습니다. 다시 시도해주세요" 오류 메시지 표시 |
| E-PP-09 | 브라우저 메모리 부족으로 처리 불가 | "파일이 너무 커서 처리할 수 없습니다. 더 작은 파일을 사용해주세요" 오류 메시지 표시 |
| E-PP-10 | 암호화 완료 후 '새 작업 시작' 클릭 | 비밀번호 입력 필드와 파일 정보 모두 초기화, 메모리 참조 해제 |
| E-PP-11 | 출력 파일명이 비어있음 | 기본값 `{원본파일명}_protected.pdf` 사용 |
| E-PP-12 | 100MB 초과 PDF | 경고 표시 후 사용자가 계속 진행 선택 가능 |

### 11.8 이미지를 PDF로 변환 Edge Cases (Image to PDF)

| # | 시나리오 | 기대 동작 |
|---|----------|-----------|
| E-IP-01 | 지원하지 않는 이미지 형식 업로드 (예: BMP, TIFF, SVG) | 파일 거부, "지원하지 않는 이미지 형식입니다 (JPG, PNG, WEBP, GIF만 지원)" 오류 표시 |
| E-IP-02 | 손상된 이미지 파일 업로드 | 해당 파일 거부, "유효하지 않은 이미지 파일입니다" 오류 표시 |
| E-IP-03 | 이미지 0개인 상태에서 변환 버튼 클릭 시도 | 변환 버튼 비활성화 |
| E-IP-04 | 이미지 1개만 업로드 | 정상 처리, 1페이지 PDF 다운로드 |
| E-IP-05 | 매우 많은 이미지 업로드 (50장 이상) | 처리 허용, 메모리 경고 표시 가능 |
| E-IP-06 | 매우 큰 이미지 파일 (단일 50MB 이상) | 경고 표시 후 사용자가 계속 진행 선택 가능 |
| E-IP-07 | 이미지 원본 크기 옵션에서 이미지마다 크기가 다를 경우 | 각 페이지 크기가 해당 이미지 원본 크기를 따름 (페이지별 크기 상이 허용) |
| E-IP-08 | 투명 배경 PNG 이미지 | 투명 배경을 흰색으로 처리하여 PDF에 배치 |
| E-IP-09 | 가로 방향(Landscape) 이미지를 A4 페이지에 배치 | 비율 유지하며 A4 가로폭 또는 세로폭 기준으로 fit 배치 |
| E-IP-10 | 출력 파일명이 비어있음 | 기본값 `images.pdf` 사용 |
| E-IP-11 | 출력 파일명에 특수문자 입력 | 파일시스템 안전 문자로 sanitize |
| E-IP-12 | 애니메이션 GIF 업로드 | 첫 번째 프레임만 사용하여 PDF에 배치, 안내 메시지 표시 |

### 정량적 기준

| 추출 기능 완성도 | 모든 AC-EX 항목 통과 |
| 삭제 기능 완성도 | 모든 AC-D 항목 통과 |
| 회전 기능 완성도 | 모든 AC-R 항목 통과 |
| 순서 변경 기능 완성도 | 모든 AC-RO 항목 통과 |

| 스토리지 잔류 없음 | localStorage/sessionStorage/IndexedDB 데이터 잔류 0건 |

| 브라우저 호환성 | Chrome, Firefox, Edge 최신 버전 동작 |

### 정성적 기준

- 비전문가 사용자가 설명 없이 8가지 기능을 완료할 수 있다
- 파일이 서버로 전송되지 않는다는 신뢰를 UI에서 전달한다
- 오류 상황에서 사용자가 다음 행동을 알 수 있는 메시지를 제공한다
- 각 기능의 차이를 사용자가 직관적으로 구분할 수 있다
- Reorder/Image to PDF 기능에서 드래그앤드롭이 자연스럽고 직관적으로 동작한다
- Password Protect 완료 후 비밀번호 분실 위험을 안내하여 사용자가 인지한다
