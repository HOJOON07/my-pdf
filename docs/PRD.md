## 1. 제품 개요

vibe-pdf는 사용자가 파일을 서버에 업로드하지 않고 브라우저에서 직접 PDF를 처리할 수 있는 클라이언트 사이드 전용 웹 애플리케이션이다. 병합(Merge), 분할(Split), 페이지 추출(Extract), 페이지 삭제(Delete), 페이지 회전(Rotate), 페이지 순서 변경(Reorder), 암호화(Password Protect), 이미지를 PDF로 변환(Image to PDF), 잠금 해제(Unlock), 문서 정보 보기(Info), 메타데이터 제거(Remove Metadata), 페이지 번호 추가(Add Page Numbers), 홀수/짝수 페이지 선택(Odd/Even) 기능을 제공한다.

| 항목 | 내용 |
|------|------|
| 제품명 | vibe-pdf |
| 플랫폼 | 웹 브라우저 (데스크톱 우선) |
| 기술 스택 | React + TypeScript + Vite + pdf-lib |
| 배포 방식 | 정적 파일 호스팅 (서버 없음) |
| MVP 기능 | PDF 병합, PDF 스플릿, PDF 페이지 추출, PDF 페이지 삭제, PDF 페이지 회전, PDF 페이지 순서 변경, PDF 암호화, 이미지를 PDF로 변환, PDF 잠금 해제, 문서 정보 보기, 메타데이터 제거, 페이지 번호 추가, 홀수/짝수 페이지 선택 (13개) |

## 2. 문제 정의

### 현재 사용자가 겪는 문제

1. **프라이버시 위협**: 기존 PDF 도구(Smallpdf, iLovePDF 등)는 파일을 서버에 업로드한다. 민감한 문서(계약서, 개인 정보 포함 문서)를 제3자 서버로 보내야 한다.
2. **불필요한 복잡성**: 많은 PDF 도구가 OCR, 압축, 서명, 변환 등 불필요한 기능으로 인터페이스가 복잡하다.
3. **속도 저하**: 서버 업로드/처리/다운로드 왕복으로 처리 시간이 길다.
4. **비용**: 대부분의 고품질 도구는 유료이거나 기능을 제한한다.
5. **페이지 단위 편집 불가**: 특정 페이지만 뽑거나, 불필요한 페이지를 제거하는 간단한 작업도 전용 소프트웨어가 필요하다.
6. **스캔 방향 오류**: 스캔된 문서의 일부 페이지가 잘못된 방향으로 저장되어 있어도 간단히 회전할 수 없다.
7. **페이지 순서 수정 불가**: PDF 내 페이지 순서가 잘못되었을 때 재배열하려면 별도 소프트웨어가 필요하다.
8. **PDF 보안 적용 불가**: 민감한 PDF를 공유할 때 비밀번호를 설정하려면 별도 소프트웨어나 서버 기반 도구가 필요하다.
9. **이미지를 PDF로 변환 불가**: 스캔 이미지나 사진을 PDF로 묶으려면 별도 소프트웨어나 서버 기반 변환 도구가 필요하다.
10. **암호화된 PDF 잠금 해제 불가**: 비밀번호를 알고 있어도 제거(잠금 해제)하려면 별도 소프트웨어가 필요하다.
11. **문서 메타데이터 확인/제거 불가**: PDF에 포함된 작성자, 소프트웨어, 생성일 등 개인정보 관련 메타데이터를 확인하거나 제거하기 어렵다.
12. **페이지 번호 수동 삽입**: 페이지 번호가 없는 PDF에 번호를 추가하려면 별도 소프트웨어가 필요하다.
13. **홀수/짝수 페이지 일괄 추출 불가**: 스캔 문서에서 홀수 또는 짝수 페이지만 빠르게 추출하는 기능이 없다.

### 핵심 통찰

"PDF에 관한 일상적인 13가지 작업 모두 서버 없이 브라우저에서 처리 가능하다. 파일이 내 장치를 떠나지 않아도 된다."

## 3. 목표

### 제품 목표

- 파일이 서버로 전송되지 않는 프라이버시 보장 PDF 도구 제공
- PDF 13가지 기능을 최소 클릭으로 완료 가능한 단순 UX 구현
- 설치 없이 브라우저에서 즉시 사용 가능한 접근성 확보

### MVP 범위 목표

- PDF 병합: 2개 이상의 PDF 파일을 하나로 합치기
- PDF 스플릿: 하나의 PDF 파일을 페이지 단위로 분리하기
- PDF 페이지 추출 (Extract): 원하는 페이지만 골라 새 PDF로 저장하기
- PDF 페이지 삭제 (Delete): 불필요한 페이지를 제거하고 저장하기
- PDF 페이지 회전 (Rotate): 특정 페이지를 90/180/270도 회전하여 저장하기
- PDF 페이지 순서 변경 (Reorder): 드래그앤드롭으로 페이지 순서를 변경하여 저장하기
- PDF 암호화 (Password Protect): user password를 설정하여 AES-256으로 암호화된 PDF 저장하기
- 이미지를 PDF로 변환 (Image to PDF): JPG/PNG 등 이미지 파일을 페이지로 배치하여 단일 PDF로 저장하기
- PDF 잠금 해제 (Unlock): 비밀번호를 알고 있는 암호화 PDF의 비밀번호를 제거하여 저장하기
- 문서 정보 보기 (Info): PDF 메타데이터(제목, 작성자, 생성일, 페이지 수, 파일 크기 등) 표시하기
- 메타데이터 제거 (Remove Metadata): PDF에 포함된 개인정보 관련 메타데이터 삭제 후 저장하기
- 페이지 번호 추가 (Add Page Numbers): PDF 각 페이지에 번호 텍스트를 삽입하여 저장하기
- 홀수/짝수 페이지 선택 (Odd/Even): PDF에서 홀수 또는 짝수 페이지만 추출하여 저장하기

## 4. 대상 사용자

### Primary User: 일반 직장인 / 사무직

- **특징**: PDF 편집 전문가가 아님. 가끔 PDF 합치기/나누기/페이지 편집이 필요함

- **니즈**: 빠르고 단순하게 처리. 파일이 외부로 나가지 않기를 원함
- **기기**: 데스크톱 / 노트북 (크롬, 엣지, 파이어폭스)

- **특징**: 비용에 민감. 무료 도구 선호
- **상황**: 과제 제출용 PDF 합치기, 포트폴리오 PDF 편집, 스캔 문서 정리, 사진을 PDF로 제출
- **니즈**: 무료, 빠름, 설치 불필요

## 5. 핵심 기능

### Feature 1: PDF 병합 (Merge)

여러 PDF 파일을 사용자가 지정한 순서대로 하나의 PDF로 합친다.

### Feature 2: PDF 스플릿 (Split)

하나의 PDF 파일을 선택한 페이지 범위 또는 각 페이지별로 분리하여 다운로드한다.

### Feature 3: PDF 페이지 추출 (Extract)

하나의 PDF 파일에서 원하는 특정 페이지만 선택하여 새로운 PDF 파일로 저장한다.

### Feature 4: PDF 페이지 삭제 (Delete)

하나의 PDF 파일에서 불필요한 특정 페이지를 제거하고 나머지 페이지로 구성된 새 PDF 파일로 저장한다.

### Feature 5: PDF 페이지 회전 (Rotate)

하나의 PDF 파일에서 특정 페이지를 선택하여 90도/180도/270도 단위로 회전하고 저장한다.

### Feature 6: PDF 페이지 순서 변경 (Reorder)

하나의 PDF 파일의 페이지 순서를 드래그앤드롭으로 변경하고 새 PDF 파일로 저장한다.

### Feature 7: PDF 암호화 (Password Protect)

암호화되지 않은 PDF 파일에 user password(열기 비밀번호)를 설정하여 AES-256으로 암호화한다.

### Feature 8: 이미지를 PDF로 변환 (Image to PDF)

JPG, PNG, WEBP, GIF 등의 이미지 파일 1개 이상을 업로드하여 각 이미지를 한 페이지로 배치한 단일 PDF 파일로 변환한다. 이미지 순서를 드래그앤드롭으로 변경할 수 있으며 페이지 크기 옵션(A4, Letter, 이미지 원본 크기)을 선택할 수 있다.

### Feature 9: PDF 잠금 해제 (Unlock)

비밀번호로 암호화된 PDF 파일에 올바른 비밀번호를 입력하여 암호화를 제거하고 열람 제한 없는 새 PDF 파일로 저장한다.

### Feature 10: PDF 문서 정보 보기 (Info)

PDF 파일의 메타데이터(제목, 작성자, 생성 소프트웨어, 생성일, 수정일)와 문서 속성(페이지 수, 파일 크기, PDF 버전, 암호화 여부)을 브라우저에서 직접 조회하여 화면에 표시한다. 파일을 다운로드하거나 저장하지 않는다.

### Feature 11: 메타데이터 제거 (Remove Metadata)

PDF 파일에 포함된 메타데이터(제목, 작성자, 생성 소프트웨어, 생성일, 수정일, 키워드 등)를 모두 제거하고 새 PDF 파일로 저장한다.

### Feature 12: 페이지 번호 추가 (Add Page Numbers)

PDF 각 페이지의 지정한 위치(하단 중앙 기본)에 페이지 번호 텍스트를 삽입하여 새 PDF 파일로 저장한다.

### Feature 13: 홀수/짝수 페이지 선택 (Odd/Even)

PDF 파일에서 홀수 페이지만 또는 짝수 페이지만 추출하여 새 PDF 파일로 저장한다.

| R-07 | 출력 파일명은 기본값 `{원본파일명}_rotated.pdf`이며 사용자가 변경할 수 있어야 한다 |
| R-08 | 처리 중 로딩 상태를 표시해야 한다 |
| R-09 | 모든 처리는 브라우저(클라이언트) 내에서만 수행되어야 한다 |

| RO-05 | 페이지 카드를 드래그앤드롭으로 순서를 변경할 수 있어야 한다 |

### 6.7 PDF 암호화 (Password Protect)

| ID | 요구사항 |
|----|----------|
| PP-01 | 사용자는 암호화되지 않은 PDF 파일 1개를 선택할 수 있어야 한다 |
| PP-02 | 파일은 드래그 앤 드롭 또는 파일 선택 다이얼로그로 추가할 수 있어야 한다 |
| PP-03 | 업로드된 PDF의 파일명과 전체 페이지 수를 표시해야 한다 |
| PP-04 | user password(열기 비밀번호) 입력 필드를 제공해야 한다 |
| PP-05 | 비밀번호 확인 입력 필드를 제공해야 하며, 두 입력값이 일치해야 암호화 버튼이 활성화된다 |
| PP-06 | 비밀번호 입력 필드는 기본적으로 마스킹(●●●) 처리되어야 하며, 사용자가 표시/숨김을 전환할 수 있어야 한다 |
| PP-07 | 비밀번호는 최소 8자 이상을 권장하며, 8자 미만 입력 시 "8자 이상 권장" 안내 메시지를 표시해야 한다 (입력 차단은 하지 않음) |
| PP-08 | 암호화 알고리즘은 AES-256을 사용해야 한다. RC4, 40-bit 등 약한 암호화는 사용하지 않는다 |
| PP-09 | owner password는 UI에 노출하지 않고 내부적으로 생성하여 적용해야 한다 |
| PP-10 | 암호화 실행 버튼 클릭 시 브라우저에서 PDF를 암호화하여 다운로드 파일로 제공해야 한다 |
| PP-11 | 출력 파일명은 기본값 `{원본파일명}_protected.pdf`이며 사용자가 변경할 수 있어야 한다 |
| PP-12 | 처리 중 로딩 상태를 표시해야 한다 |
| PP-13 | 모든 처리는 브라우저(클라이언트) 내에서만 수행되어야 한다. 파일과 비밀번호는 서버로 전송되지 않아야 한다 |
| PP-14 | 처리 완료 후 비밀번호 데이터의 메모리 참조를 해제해야 한다 |
| PP-15 | 비밀번호 및 파일 데이터를 localStorage, sessionStorage, IndexedDB에 저장해서는 안 된다 |
| PP-16 | 이미 암호화된 PDF를 업로드하면 오류 메시지를 표시하고 처리를 거부해야 한다 |

### 6.8 이미지를 PDF로 변환 (Image to PDF)

| IP-02 | 파일은 드래그 앤 드롭 또는 파일 선택 다이얼로그로 추가할 수 있어야 한다 |

## 8. 제외 범위 (Out of Scope)

MVP에서 명시적으로 제외하는 기능:

- OCR (텍스트 인식)
- PDF 압축 / 최적화
- owner password UI 노출 및 사용자 직접 설정
- 기존 암호화 PDF 재암호화 (비밀번호 변경)
- 권한 세부 설정 (인쇄 제한, 복사 제한 등)
- 일괄 암호화 (여러 파일 동시 암호화)
- 워터마크 추가 / 제거
- PDF → 이미지/Word 변환
- 클라우드 저장 (Google Drive, Dropbox 등)
- 사용자 계정 / 인증
- 히스토리 / 최근 파일
- PDF 미리보기 썸네일 (Reorder 탭 제외)
- 주석 / 서명
- SVG, BMP, TIFF 등 비주류 이미지 형식 (Image to PDF)
- 이미지 편집 (크롭, 필터, 보정 등)
- 개별 페이지 여백/마진 설정
- 페이지 번호 폰트/크기/색상 커스텀 (Add Page Numbers MVP 기준)
- 로마 숫자, 문자 등 번호 형식 (Add Page Numbers MVP 기준, 숫자만 지원)
- 메타데이터 선택적 제거 (Remove Metadata는 전체 일괄 제거만 지원)

## 10. Acceptance Criteria

### 10.1 PDF 병합 AC

- [ ] AC-M-01: 사용자가 2개의 PDF를 추가하고 병합 버튼을 클릭하면 단일 PDF가 다운로드된다
- [ ] AC-M-02: 사용자가 3개의 PDF를 추가하고 순서를 변경한 후 병합하면 변경된 순서대로 합쳐진다
- [ ] AC-M-03: 파일 목록에서 특정 파일 제거 후 병합하면 해당 파일은 포함되지 않는다
- [ ] AC-M-04: 출력 파일명을 `report.pdf`로 변경하면 `report.pdf`로 다운로드된다
- [ ] AC-M-05: 파일 목록에 각 PDF의 파일명과 페이지 수가 표시된다
- [ ] AC-M-06: 병합 처리 중 로딩 인디케이터가 표시되고 완료 시 사라진다
- [ ] AC-M-07: PDF가 아닌 파일(.jpg, .docx 등)을 추가하려 하면 오류 메시지가 표시되고 목록에 추가되지 않는다
- [ ] AC-M-08: 파일 1개만 있는 상태에서 병합 버튼은 비활성화(disabled) 상태이다
- [ ] AC-M-09: 병합 완료 후 '새 작업 시작' 버튼을 클릭하면 파일 목록이 초기화된다

### 10.2 PDF 스플릿 AC

- [ ] AC-S-01: PDF를 업로드하면 파일명과 전체 페이지 수가 표시된다
- [ ] AC-S-02: '모든 페이지 개별 분리' 모드 선택 후 실행하면 페이지 수만큼의 PDF가 ZIP으로 다운로드된다
- [ ] AC-S-03: 범위 `1-3`을 입력하면 1, 2, 3페이지로 구성된 PDF 1개가 다운로드된다

### 10.13 홀수/짝수 페이지 선택 AC (Odd/Even)

### 11.11 메타데이터 제거 Edge Cases (Remove Metadata)

### 정성적 기준

- 비전문가 사용자가 설명 없이 13가지 기능을 완료할 수 있다
- 파일이 서버로 전송되지 않는다는 신뢰를 UI에서 전달한다
- 오류 상황에서 사용자가 다음 행동을 알 수 있는 메시지를 제공한다
- 각 기능의 차이를 사용자가 직관적으로 구분할 수 있다
- Reorder/Image to PDF 기능에서 드래그앤드롭이 자연스럽고 직관적으로 동작한다
- Password Protect / Unlock 완료 후 비밀번호 관련 보안 안내를 사용자가 인지한다
