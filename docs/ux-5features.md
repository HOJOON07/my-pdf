## 2. Feature 9: PDF 잠금 해제 (Unlock)

### 2.1 기능 정의
- 암호화된 PDF에 올바른 비밀번호를 입력하여 암호화를 제거한 새 PDF 저장
- 비밀번호 입력 UI: Password Protect 탭의 `PasswordInput` 컴포넌트 재사용
- 처리: qpdf WASM `--decrypt` (encrypt.worker.ts 재활용)
- WASM 초기화 상태 존재 (최초 1회, Password Protect와 동일)
- 암호화되지 않은 PDF 업로드 시 "이미 잠금 해제된 파일" 안내
- 출력 파일명 기본값: `{원본파일명}_unlocked.pdf`

### 2.2 사용자 플로우
```
[파일 업로드] → [암호화 여부 자동 감지]
  → 암호화된 PDF: 비밀번호 입력 섹션 표시
  → 암호화 안 된 PDF: 안내 배너 표시 (이미 잠금 해제된 파일)
→ [비밀번호 입력] → [잠금 해제하기 버튼 클릭]
→ [WASM 초기화 (최초 1회)] → [처리 중]
→ [완료: 자동 다운로드] 또는 [에러: 잘못된 비밀번호 / 처리 실패]
```

### 2.3 화면 목록
| 화면 ID | 설명 |
|---------|------|
| UL-S1 | 초기 상태 (파일 없음) |
| UL-S2 | 암호화된 PDF 업로드 완료 — 비밀번호 입력 표시 |
| UL-S3 | 암호화되지 않은 PDF 업로드 — 안내 배너 |
| UL-S4 | WASM 초기화 중 (최초 1회) |
| UL-S5 | 처리 중 |
| UL-S6 | 완료 |
| UL-S7 | 에러 (잘못된 비밀번호) |
| UL-S8 | 에러 (처리 실패 / 손상 파일) |

## 3. Feature 10: PDF 문서 정보 보기 (Info)

### 3.1 기능 정의
- PDF 파일의 메타데이터와 문서 속성을 화면에 표시 (읽기 전용, 다운로드 없음)
- 표시 항목: 파일명, 파일 크기, 페이지 수, PDF 버전, 암호화 여부, 제목, 작성자, 생성 소프트웨어(Creator/Producer), 생성일, 수정일, 키워드
- 빈 항목: "-"로 표시
- 클립보드 복사 버튼 제공
- 암호화된 PDF: 메타데이터 읽기 불가 → 접근 제한 안내

### 3.2 사용자 플로우
```
[파일 업로드] → [즉시 정보 파싱 + 표시]
  → 암호화된 PDF: 기본 정보만 표시 + 메타데이터 접근 제한 안내
  → 일반 PDF: 전체 정보 표시
→ [클립보드 복사 (선택)] → [다른 파일 선택 또는 X 버튼]
```
- 다운로드 버튼 없음 (읽기 전용)
- "새 작업 시작" 버튼 대신 파일 카드의 X 버튼으로 초기화

### 3.5 클립보드 복사 기능
- "정보 복사" 버튼 클릭 시 `navigator.clipboard.writeText()` 호출
- 복사되는 텍스트 형식:
  ```
  파일명: report.pdf
  파일 크기: 2.4 MB
  페이지 수: 12페이지
  PDF 버전: 1.7
  암호화: 잠금 해제됨
  ---
  제목: 보고서 제목
  작성자: 홍길동
  ...
  ```
- 복사 성공: 버튼 텍스트 → "복사됨!" (2초 후 원래대로)
- 클립보드 API 미지원: 버튼 비활성화 + `title="클립보드 복사가 지원되지 않는 브라우저예요"` tooltip

### 3.6 긴 메타데이터 처리
- 500자 초과: `truncate` 처리 + "전체 보기" 토글 버튼
- 특수문자: 이스케이프 없이 그대로 표시

| 상태 | 버튼 | 입력 | 특이사항 |
|------|------|------|---------|

## 11. 접근성 고려사항

- 모든 신규 페이지: `<main>` 내 `<section>` 구조, 각 섹션에 `<h2>` 레이블
- 암호화 뱃지/배너: `role="status"` (안내) 또는 `role="alert"` (에러) 구분
- Unlock 비밀번호 필드: PasswordInput 컴포넌트 접근성 그대로 상속
- MetadataPanel 정보 테이블: `dl/dt/dd` 또는 `grid` — 스크린 리더 읽기 순서 보장
- PagePositionPicker: `fieldset` + `legend` + `input[type="radio"]`
- OddEven 라디오 카드: SplitModeSelector 패턴 (`fieldset` + `legend`)
- 처리 완료 시 포커스: 완료 카드의 첫 번째 버튼으로 이동
- 처리 에러 시 포커스: 에러 카드로 이동 (`role="alert"` 자동 읽힘)
