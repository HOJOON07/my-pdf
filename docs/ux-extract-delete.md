# UX/UI 설계 - PDF 추출(Extract) / 삭제(Delete) 화면

> 버전: 1.1
> 기준 문서: PRD v1.1, PDF_EXTRACT_DELETE_SPEC.md
> 작성일: 2026-03-19
> 기존 Merge / Split UI 패턴과 일관성 유지. 서버 없는 브라우저 기반 처리.

---

## 1. 정보 구조 (IA)

```
vibe-pdf
├── 병합 (Merge)          ← 기존
├── 분할 (Split)          ← 기존
├── 추출 (Extract)        ← 신규
└── 삭제 (Delete)         ← 신규
```

탭 네비게이션에 2개 탭 추가. 탭 순서: 병합 → 분할 → 추출 → 삭제

---

## 2. 기능 정의

### Extract (추출)
- PDF에서 원하는 페이지만 골라 새 PDF로 저장
- 결과: 선택한 페이지들로 구성된 단일 PDF 파일 1개 다운로드
- 출력 파일명: 기본값 `extracted.pdf`, 사용자 변경 가능
- Split과의 차이: Split은 전체를 여러 조각으로 나누는 것, Extract는 원하는 페이지만 골라 단일 PDF 생성

### Delete (삭제)
- PDF에서 삭제할 페이지를 지정하고 나머지를 새 PDF로 저장
- 핵심 UX: 입력 중 "삭제 후 남은 페이지 수" 실시간 피드백
- 결과: 지정 페이지를 제외한 단일 PDF 파일 1개 다운로드
- 출력 파일명: 기본값 `{원본파일명}_deleted.pdf`, 사용자 변경 가능

**Extract와 Delete의 구조는 거의 동일하나 Delete는 "남은 페이지" 피드백 섹션이 추가된다.**

## 4. 주요 사용자 플로우

### 4.1 Extract 플로우

```
[초기] 파일 없음
  → FileDropZone 표시 (파일 1개)
  → 파일 드롭 or 클릭 선택
[파일 로드됨]
  → 파일 정보 카드 표시 (파일명, 전체 페이지 수)
  → 추출할 페이지 범위 입력 필드
  → 출력 파일명 입력 (기본값: extracted.pdf)
  → 유효한 범위 입력 시 "PDF 추출하기 →" 버튼 활성화
  → 버튼 클릭
[처리 중]
  → 스피너 + "추출 중..." 텍스트, 모든 입력 disabled
[완료]
  → 성공 카드 (초록 배경, CheckCircle 아이콘)
  → "새 작업 시작" 버튼으로 초기화
[오류]
  → 인라인 에러 배너 (빨간 배경, AlertCircle 아이콘)
  → 입력 상태 유지, 재시도 가능
```

### 4.2 Delete 플로우

```
[초기] 파일 없음
  → FileDropZone 표시 (파일 1개)
  → 파일 드롭 or 클릭 선택
[파일 로드됨]
  → 파일 정보 카드 표시 (파일명, 전체 페이지 수)
  → 삭제할 페이지 범위 입력 필드
  → 입력 중 실시간: "삭제 후 남은 페이지: N페이지" 피드백 표시
  → 출력 파일명 입력 (기본값: {원본파일명}_deleted.pdf)
  → 유효하고 남은 페이지 >= 1이면 "PDF 삭제하기 →" 버튼 활성화
  → 버튼 클릭
[처리 중]
  → 스피너 + "삭제 중..." 텍스트, 모든 입력 disabled
[완료]
  → 성공 카드 (초록 배경, CheckCircle 아이콘)
  → "새 작업 시작" 버튼으로 초기화
[오류]
  → 인라인 에러 배너 + 입력 상태 유지
```

## 7. RemainingPagesInfo 컴포넌트 상세 (Delete 전용)

삭제 탭의 핵심 피드백 컴포넌트. PageRangeInput 아래, OutputNameInput 위에 위치.

### 표시 조건 및 스타일

| 조건 | 표시 내용 | 스타일 |
|------|----------|--------|
| 범위 미입력 | 렌더링 안 함 | — |
| 유효 입력 + 남은 페이지 >= 1 | "삭제 후 남은 페이지: N페이지" | `bg-blue-50 border-blue-200 text-blue-800` |
| 유효 입력 + 남은 페이지 = 0 | "모든 페이지를 삭제할 수 없어요. 일부 페이지는 남겨야 합니다." | `bg-amber-50 border-amber-200 text-amber-800` + `role="alert"` |
| 범위 입력 오류 | 렌더링 안 함 (PageRangeInput 인라인 에러로 처리) | — |

### Props

```typescript
interface RemainingPagesInfoProps {
  remainingCount: number | null  // null이면 렌더링 안 함
}
```

## 8. 훅 상태 구조

### 8.1 useExtract

```typescript
interface ExtractState {
  file: { file: File; pageCount: number } | null
  rangeInput: string
  rangeError: string | null
  outputName: string          // 기본값: 'extracted.pdf'
  status: ProcessingStatus    // 'idle' | 'processing' | 'done' | 'error'
  error: string | null
  canExecute: boolean         // rangeError === null && rangeInput.trim() !== '' && file !== null
}
```

### 8.2 useDelete

```typescript
interface DeleteState {
  file: { file: File; pageCount: number } | null
  rangeInput: string
  rangeError: string | null
  remainingCount: number | null  // 실시간 계산: null = 미입력 또는 오류
  outputName: string             // 기본값: '{원본파일명}_deleted.pdf'
  status: ProcessingStatus
  error: string | null
  canExecute: boolean            // rangeError === null && rangeInput !== '' && remainingCount >= 1
}
```

`remainingCount` 계산: rangeInput 변경 시마다 `parsePageRanges` 결과로 `totalPages - deletedIndices.length` 실시간 계산. `rangeError`가 있으면 null 유지.

## 9. 상태별 UX 정의

### 9.1 Extract 상태표

| 상태 | 조건 | 버튼 | 피드백 |
|------|------|------|--------|
| 초기 | file === null | 미표시 | FileDropZone만 |
| 파일 있음 + 범위 미입력 | rangeInput === '' | 비활성 | 에러 없음 |
| 파일 있음 + 범위 유효 | rangeError === null, input 있음 | 활성 | 힌트 텍스트 |
| 파일 있음 + 범위 오류 | rangeError !== null | 비활성 | 인라인 에러 (빨간) |
| 처리 중 | status === 'processing' | 스피너 + 비활성 | 모든 입력 disabled |
| 완료 | status === 'done' | 미표시 | 성공 카드 |
| 오류 | status === 'error' | 활성 복귀 | 에러 배너 + 재시도 가능 |

### 9.2 Delete 상태표

| 상태 | 조건 | 버튼 | 피드백 |
|------|------|------|--------|
| 초기 | file === null | 미표시 | FileDropZone만 |
| 파일 있음 + 범위 미입력 | rangeInput === '' | 비활성 | RemainingPagesInfo 없음 |
| 유효 + 남은 >= 1 | remainingCount >= 1 | 활성 | RemainingPagesInfo 파란색 |
| 유효 + 남은 = 0 | remainingCount === 0 | 비활성 | RemainingPagesInfo 주황색 경고 |
| 범위 오류 | rangeError !== null | 비활성 | 인라인 에러, RemainingPagesInfo 없음 |
| 처리 중 | status === 'processing' | 스피너 + 비활성 | 모든 입력 disabled |
| 완료 | status === 'done' | 미표시 | 성공 카드 |
| 오류 | status === 'error' | 활성 복귀 | 에러 배너 + 재시도 가능 |

## 12. 접근성 / 사용성 고려사항

1. **탭 순서 (Tab key)**: FileDropZone → 파일 제거 X버튼 → 범위 입력 → 출력 파일명 입력 → 실행 버튼
2. **aria-label**: 파일 제거 버튼 `${file.name} 제거` — 기존 SplitPage 패턴 동일
3. **aria-invalid + aria-describedby**: 범위 입력 오류 시 적용, 에러 메시지 id 연결
4. **role="alert"**: 에러 배너, 전체 삭제 경고(RemainingPagesInfo 주황) 모두 적용
5. **disabled 상태**: 처리 중 모든 인터랙티브 요소 `disabled` + `opacity-50` — 기존 패턴 동일
6. **Enter/Space 키**: FileDropZone 키보드 접근 지원 — 기존 컴포넌트 그대로
7. **색상 대비**: 남은 페이지 정상 피드백 blue, 경고 amber — WCAG AA 기준 충족
8. **파일명 sanitize**: 출력 파일명 입력 시 파일시스템 안전 문자로 처리 (PRD Edge Case E-M-08 대응)
