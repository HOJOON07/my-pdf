## 1. 테스트 범위

### 1.1 테스트 대상 기능

| 기능 | 범위 |
|------|------|
| PDF 병합 (Merge) | 파일 추가, 순서 변경, 파일 제거, 출력 파일명 설정, 병합 실행, 다운로드 |
| PDF 스플릿 (Split) | 파일 추가, 페이지 수 표시, 분할 모드 선택, 범위 입력, 실행, 다운로드 |
| PDF 추출 (Extract) | 파일 추가, 페이지 범위 입력, 출력 파일명, 추출 실행, 단일 PDF 다운로드 |
| PDF 삭제 (Delete) | 파일 추가, 삭제 범위 입력, 남은 페이지 미리보기, 삭제 실행, 다운로드 |
| PDF 회전 (Rotate) | 파일 추가, 페이지 범위 입력(선택), 각도 선택, 회전 실행, 단일 PDF 다운로드 |
| PDF 순서 변경 (Reorder) | 파일 추가, 페이지 카드 목록, 드래그앤드롭, Reset, 순서 변경 PDF 다운로드 |
| PDF 암호화 (Encrypt) | 파일 추가, 비밀번호 입력, 비밀번호 확인, 출력 파일명, 암호화 실행, 다운로드 |
| 이미지→PDF (Image to PDF) | 이미지 업로드, 포맷 검증, 순서 변경, 페이지 크기 선택, 변환 실행, 다운로드 |
| 문서 정보 (Info) | PDF 업로드, 자동 분석, 메타데이터 표시, 암호화 처리, 정보 복사 |
| 메타데이터 제거 (Remove Metadata) | 파일 업로드, 메타데이터 미리보기, 제거 실행, 다운로드 |
| 페이지 번호 추가 (Add Page Numbers) | 파일 업로드, 위치 선택, 시작 번호, 번호 추가, 다운로드 |
| 홀수/짝수 선택 (Odd/Even) | 파일 업로드, 모드 선택, 페이지 수 힌트, 추출 실행, 다운로드 |
| 공통 입력 검증 | 파일 타입 검사, 파일 크기 정책, 비활성화 버튼 상태 |
| 에러 처리 | 암호화 파일, 손상 파일, 잘못된 범위 입력, 메모리 한계 |
| 다운로드 결과 검증 | PDF 무결성, 페이지 순서, ZIP 구성, 파일명 |
| 브라우저/환경 호환성 | Chrome, Firefox, Edge, Safari, 모바일 |
| 보안/프라이버시 | 네트워크 전송 없음 확인 |

### 1.2 구현 환경 특이사항 (v2.0 추가)

| 항목 | 명세 기준 | 실제 구현 | 비고 |
|------|----------|----------|------|
| 파일 크기 제한 (reject) | 100MB | **400MB** | validate.ts MAX_FILE_SIZE |
| 파일 크기 경고 (warn) | 50MB | **200MB** | validate.ts WARN_FILE_SIZE |
| Extract 출력 | Split과 동일 | **항상 단일 PDF** | 범위 다중 입력 시에도 단일 파일로 병합 |
| Delete 기본 파일명 | 미정 | `{원본파일명}_deleted.pdf` | useDelete.ts |
| Extract 기본 파일명 | 미정 | `extracted.pdf` | useExtract.ts |
| 파일명 .pdf 미입력 | 미정 | **자동으로 .pdf 확장자 추가** | useExtract/useDelete 공통 |
| 파일 타입 검증 | MIME AND 확장자 | **MIME OR 확장자** | validate.ts 조건 주의 |
| 암호화 라이브러리 | @cantoo/qpdf WASM | **pdf-lib-plus-encrypt** (순수 JS) | Web Worker 미사용, 메인 스레드 처리 |
| Encrypt 파일 크기 거부 | 100MB | **100MB** (ENCRYPT_MAX 별도 상수) | Encrypt 전용 제한 |
| Image to PDF 포맷 변환 | — | WebP/GIF/BMP → Canvas → PNG 경유 | HEIC/TIFF/SVG 미지원 |
| Unlock 기능 | 구현 예정 (PDF_5FEATURES_SPEC) | **미구현** | App.tsx 탭 없음, 구현 파일 없음 |
| 탭 레이아웃 변경 | overflow-x-auto → | **flex-wrap gap-1** | App.tsx:50, 12탭 대응 |

### 3.3 PDF 추출 (Extract) — v2.0 신규

> Extract는 지정한 페이지들을 **항상 단일 PDF 파일**로 추출한다.
> Split의 "범위 지정 분할"과 달리, 다중 범위 입력도 하나의 파일로 합쳐서 출력한다.

#### 3.3.1 정상 케이스

| ID | 시나리오 | 입력 | 기대 결과 |
|----|----------|------|-----------|
| TC-E-001 | 파일 업로드 후 정보 표시 | 10페이지 PDF 업로드 | 파일명과 "전체 10페이지" 표시, DropZone 사라짐 |
| TC-E-002 | 단일 페이지 추출 | 5페이지 PDF, `3` 입력 | 1페이지 PDF `extracted.pdf` 다운로드 |
| TC-E-003 | 연속 범위 추출 | 10페이지 PDF, `2-5` 입력 | 4페이지 PDF `extracted.pdf` 다운로드 |
| TC-E-004 | 복합 범위 추출 (단일 파일) | 10페이지 PDF, `1-3, 7` 입력 | 4페이지 PDF 단일 파일 다운로드 (Split과 다름) |
| TC-E-005 | 출력 파일명 변경 | `report_extract.pdf` 입력 후 추출 | `report_extract.pdf`로 다운로드 |

## 8. 회귀 테스트 체크리스트

### 8.1 병합 핵심 시나리오

- [ ] 2개 PDF 병합 → 단일 PDF 다운로드 → 페이지 수 합산 정확

## 10. Extract vs Split 동작 차이 검증

> Extract와 Split은 유사해 보이지만 핵심 동작이 다르다. 혼동을 방지하기 위해 아래 케이스를 명시적으로 검증한다.

| ID | 입력 | Extract 결과 | Split 결과 |
|----|------|------------|-----------|
| CMP-001 | 10페이지 PDF, `1-3` 입력 | 3페이지 단일 PDF | 3페이지 단일 PDF (동일) |
| CMP-002 | 10페이지 PDF, `1-3, 7` 입력 | **4페이지 단일 PDF** | **2개 파일** (3페이지 + 1페이지) ZIP |
| CMP-003 | 10페이지 PDF, `1, 3, 5` 입력 | **3페이지 단일 PDF** | **3개 파일** 각 1페이지 ZIP |
| CMP-004 | 결과 파일 형식 | 항상 단일 PDF | 결과 수에 따라 PDF 또는 ZIP |
