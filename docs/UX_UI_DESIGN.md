# UX/UI 설계: vibe-pdf — 브라우저 기반 PDF Merge / Split MVP

**버전**: 1.0
**작성일**: 2026-03-18
**작성자**: UX/UI Designer
**기반 문서**: PRD v1.0, PDF_DOMAIN_SPEC v1.0.0

## 1. 정보 구조 (IA)

```
vibe-pdf
├── 앱 헤더
│   ├── 로고 / 서비스명
│   ├── 신뢰 배지 ("파일이 내 기기를 떠나지 않아요")
│   └── 기능 탭 내비게이션
│       ├── 병합 (Merge)
│       └── 분할 (Split)
│
├── [병합 탭] MergePage
│   ├── 파일 업로드 존 (DropZone)
│   ├── 파일 목록 (FileList)
│   │   ├── 파일 아이템 (FileItem) × N
│   │   │   ├── 순서 번호
│   │   │   ├── 파일명
│   │   │   ├── 페이지 수
│   │   │   ├── 파일 크기
│   │   │   └── 삭제 버튼
│   │   └── 파일 추가 버튼 (+ 파일 더 추가)
│   ├── 출력 파일명 입력 (OutputFilename)
│   └── 실행 영역
│       ├── 병합 버튼 (MergeButton)
│       ├── 로딩 인디케이터
│       └── 완료/에러 피드백
│
└── [분할 탭] SplitPage
    ├── 파일 업로드 존 (DropZone)
    ├── 파일 정보 표시 (FileInfo)
    │   ├── 파일명
    │   └── 전체 페이지 수
    ├── 분할 모드 선택 (SplitModeSelector)
    │   ├── 모든 페이지 개별 분리
    │   └── 범위 지정 분할
    │       └── 범위 입력 필드 (PageRangeInput)
    └── 실행 영역
        ├── 분할 버튼 (SplitButton)
        ├── 로딩 인디케이터
        └── 완료/에러 피드백
```

## 3. 주요 사용자 플로우

### 3.1 PDF 병합 플로우 (Happy Path)

```
[앱 로드] → [병합 탭 선택 (기본)]
    → [파일 드래그앤드롭 또는 클릭하여 선택]
    → [파일 목록 표시: 파일명, 페이지 수, 크기]
    → (선택) [드래그로 순서 변경]
    → (선택) [파일 개별 삭제]
    → (선택) [출력 파일명 변경]
    → [병합 버튼 클릭]
    → [로딩 상태: 처리 중]
    → [자동 다운로드: merged.pdf]
    → [완료 메시지 + 새 작업 시작 버튼]
```

### 3.2 PDF 분할 플로우 (Happy Path)

```
[앱 로드] → [분할 탭 클릭]
    → [파일 드래그앤드롭 또는 클릭하여 선택]
    → [파일 정보 표시: 파일명, 전체 페이지 수]
    → [분할 모드 선택]
    │   ├── [모든 페이지 개별 분리 선택] → [분할 버튼 활성화]
    │   └── [범위 지정 선택] → [범위 입력 필드 표시] → [범위 입력] → [실시간 검증]
    → [분할 버튼 클릭]
    → [로딩 상태: 처리 중]
    → [다운로드: 단일 PDF 또는 ZIP]
    → [완료 메시지 + 새 작업 시작 버튼]
```

## 4. 화면별 레이아웃 설명

### 4.1 공통 레이아웃

```
┌────────────────────────────────────────────┐
│ [로고: vibe-pdf]   [파일이 내 기기를 떠나지 않아요 ⚡] │  ← 헤더 (sticky)
├────────────────────────────────────────────┤
│         [ 병합 (Merge) ] [ 분할 (Split) ]         │  ← 탭 내비게이션
├────────────────────────────────────────────┤
│                                            │
│              [탭 콘텐츠 영역]                │
│                                            │
└────────────────────────────────────────────┘
```

- 최대 너비: 800px, 중앙 정렬
- 배경: 밝은 중성 색상 (흰색 or 연한 회색)
- 헤더는 항상 상단에 고정

---

### 4.2 병합 탭 — 초기 (S-01)

```
┌────────────────────────────────────────────┐
│ [병합 탭 활성] [ 분할 탭 ]                    │
├────────────────────────────────────────────┤
│                                            │
│  ┌──────────────────────────────────────┐  │
│  │                                      │  │
│  │   [PDF 아이콘]                         │  │
│  │   PDF 파일을 여기에 끌어다 놓거나          │  │
│  │   [파일 선택] 버튼을 클릭하세요            │  │
│  │                                      │  │
│  │   * 여러 파일을 동시에 선택할 수 있어요     │  │
│  └──────────────────────────────────────┘  │
│  (DropZone — 점선 테두리, hover 시 강조)       │
│                                            │
└────────────────────────────────────────────┘
```

- 병합 탭이 기본 선택 상태
- DropZone은 클릭 또는 드래그앤드롭 모두 지원
- 탭 아래 작은 글씨로 기능 설명 한 줄 제공 가능

---

### 4.3 병합 탭 — 파일 추가됨 (S-02)

```
┌────────────────────────────────────────────┐
│ [병합 탭 활성] [ 분할 탭 ]                    │
├────────────────────────────────────────────┤
│                                            │
│  파일 목록 (2개 이상 필요)                     │
│  ┌──────────────────────────────────────┐  │
│  │ ⠿  1  📄 계약서_최종.pdf    12쪽  2.3MB  ✕│  │
│  │ ⠿  2  📄 첨부서류.pdf      4쪽   0.8MB  ✕│  │
│  │ ⠿  3  📄 부록.pdf          7쪽   1.2MB  ✕│  │
│  └──────────────────────────────────────┘  │
│  [+ 파일 더 추가]                             │
│                                            │
│  출력 파일명                                  │
│  ┌──────────────────────────────────────┐  │
│  │ merged.pdf                            │  │
│  └──────────────────────────────────────┘  │
│                                            │
│         [ PDF 병합하기  →  ]                 │  ← 주요 CTA
│                                            │
└────────────────────────────────────────────┘
```

- ⠿ 아이콘은 드래그 핸들 (grab cursor)
- 파일 아이템은 드래그로 재정렬 가능
- 파일 1개만 있을 때 병합 버튼은 비활성화, 안내 문구 표시
- 출력 파일명은 편집 가능한 텍스트 입력 필드

---

### 4.4 병합 탭 — 처리 중 (S-03)

```
┌────────────────────────────────────────────┐
│  ...파일 목록 (편집 불가 상태)...               │
│                                            │
│     [  ◌ PDF 병합 중... 잠시만 기다려 주세요  ]  │
│                                            │
└────────────────────────────────────────────┘
```

- 모든 입력 및 버튼 비활성화 (disabled)
- 버튼에 스피너 아이콘 + 텍스트 변경
- 파일 목록, DropZone 인터랙션 차단

---

### 4.5 병합 탭 — 완료 (S-04)

```
┌────────────────────────────────────────────┐
│                                            │
│  ┌──────────────────────────────────────┐  │
│  │  ✓  병합 완료!                          │  │
│  │  merged.pdf 다운로드가 시작되었습니다.       │  │
│  │                                      │  │
│  │  [다시 다운로드]   [새 작업 시작]           │  │
│  └──────────────────────────────────────┘  │
│                                            │
└────────────────────────────────────────────┘
```

- 완료 메시지는 배너 형태로 상단 표시
- "새 작업 시작" 클릭 시 파일 목록 초기화, 초기 상태로 복귀
- "다시 다운로드"는 동일 결과 파일 재다운로드

## 6. 상태별 UX 정의

### 6.1 초기 상태

- DropZone이 중앙에 넓게 표시
- 병합 탭이 기본 선택
- CTA 버튼 없음 (파일 없으므로)
- 신뢰 배지 헤더에 항상 노출

### 6.2 파일 업로드 후

**병합:**
- 파일 목록 렌더링 (순서 번호, 파일명, 페이지 수, 크기, 삭제 버튼)
- 파일 1개면 병합 버튼 비활성화 + 안내 문구 "2개 이상 파일이 필요합니다"
- 파일 2개 이상이면 병합 버튼 활성화
- DropZone이 "파일 더 추가" 작은 버튼으로 축소

**분할:**
- 파일 정보 (파일명, 전체 페이지 수) 표시
- 분할 모드 선택 UI 표시
- 모드 미선택 시 분할 버튼 비활성화

### 6.3 처리 중

- 버튼 텍스트 변경: "PDF 병합 중..." / "PDF 분할 중..."
- 버튼에 스피너 아이콘 추가
- 파일 목록, 모드 선택, 입력 필드 전부 disabled
- DropZone 비활성화
- 포커스 버튼에 고정 (접근성)

### 6.4 성공

- 자동 다운로드 트리거
- SuccessBanner 표시: 완료 메시지 + [다시 다운로드] + [새 작업 시작]
- 배너는 닫기 버튼 제공
- "새 작업 시작" 클릭 시 전체 초기화

### 6.5 실패

- ErrorBanner 표시: 에러 유형별 메시지 + [확인] 버튼
- 파일 목록 및 설정은 그대로 유지 (사용자가 수정 후 재시도 가능)
- 버튼 활성화 상태로 복원
- 파일별 에러(특정 파일 문제)는 해당 FileItem에 인라인 표시 가능

## 8. 접근성 / 사용성 고려사항

### 8.1 키보드 내비게이션

- Tab 키로 모든 인터랙티브 요소 포커스 이동 가능
- Enter / Space로 버튼, 라디오 버튼 활성화
- 파일 아이템 재정렬: 포커스 후 Space로 선택, 방향키로 위아래 이동, Space로 확정 (drag-free 대안)
- Escape로 에러/완료 배너 닫기

### 8.2 스크린 리더 지원

- DropZone에 `role="button"` 및 `aria-label` 적용
- 드래그앤드롭 진행 상태를 `aria-live` 영역으로 알림
- 처리 중 상태 변화를 `aria-live="polite"`로 선언
- 에러 메시지를 `role="alert"`로 선언하여 즉시 읽힘
- 완료 메시지도 `aria-live="polite"` 적용
- 파일 삭제 버튼: `aria-label="{파일명} 제거"`

### 8.3 색상 및 대비

- 에러 상태: 빨간색 계열 + 아이콘 조합 (색맹 사용자 고려)
- 성공 상태: 초록색 계열 + 아이콘 조합
- 비활성 버튼: 충분한 대비비 유지 (WCAG AA 기준)
- 텍스트와 배경 대비비 4.5:1 이상 유지

### 8.4 포커스 관리

- 에러 발생 시 포커스를 에러 배너로 이동
- 완료 배너 표시 시 포커스를 배너 내 첫 버튼으로 이동
- 처리 중 상태: 포커스 트랩 없음, 키보드 이동은 허용하되 실행 차단

### 8.5 반응형 / 모바일 고려사항

- 최소 지원 뷰포트: 375px
- 모바일에서 DropZone은 "파일 선택" 버튼 중심으로 단순화 (드래그앤드롭 불필요)
- 파일 아이템의 드래그 핸들은 터치 친화적 크기 (44x44px 이상)
- 분할 모드 선택은 큰 터치 타겟의 라디오 버튼 또는 카드 선택 방식 적용
- 완료/에러 배너는 전체 너비로 하단 또는 상단 표시

### 8.6 파일 처리 투명성

- 파일이 서버로 전송되지 않는다는 사실을 헤더 배지뿐 아니라 DropZone 하단에도 한 줄로 명시 가능
  - 예: "업로드된 파일은 이 브라우저 탭 안에서만 처리됩니다"
- 처리 중 메시지에도 "브라우저에서 처리 중" 명시 가능

### 8.7 오류 회복성

- 에러 발생 시 파일 목록 및 설정 유지 (사용자 재작업 최소화)
- 파일 1개 조건 미충족: 에러 배너 대신 버튼 비활성화 + 인라인 안내 문구 (비교적 부드러운 피드백)
- 입력 유효성 오류(페이지 범위): submit 전 실시간 검증, 인라인 표시
- 처리 오류(암호화, 손상): 에러 배너 표시 후 재시도 가능 상태 복원
